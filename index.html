gank项目总结

项目名称：gank手机端项目

项目搭建与运行

1，运用vue-cli脚手架搭建项目 

npm install -g vue-cli
vue init webpack vue-sample

2，安装依赖包 

npm install stylus --save
npm install stylus-loader --save
npm install vue-resource --save
npm install vuex --save

项目思路以及框架搭建

要想写一个完整的项目首先需要把项目的整体框架搭建出来，然后再一点一点的往页面里面填充东西，这样一步一步的进行，当哪一点遇到问题时可以很容易的找到。
我写的这个gank项目只有三个页面，整体思路是一个标题栏（header），标题栏有一个图标点击一下会出现一个弹窗，标题栏下面时菜单栏（menu）,菜单栏的内容有 福利，Android，ios，下面就是这三个页面
因为不管点击那个页面标题栏和菜单栏都会出现，所以把他们两个设置成公共组件

Vue.js

搭建环境与运行项目

1, 想要运行一个vue项目首先需要安装git和node.js，node和git都是下载安装包直接下一步下一步安装就可以，记得如果这两个软件安装好后如果给他们改变了位置记得要从新配置环境变量啊。
2,然后安装 vue-cli 

npm install -g vue-cli

3,在你系统的某个盘里创建一个文件夹，为了存放以后你的vue项目代码我创建的是vue_gh，创建一个test项目 运行命令 

vue init webpack test

运行时一路回车等他结束会发现vue-gh文件夹中多了一个test文件夹
4，在dos命令行中进入项目文件夹 

cd vue-gh/test

5,安装依赖包 

npm install

6,启动项目 

npm run dev

vue.js的核心是允许采用简介的模板语法来声明式地将数据渲染进Dom的系统

Vue生命周期

一个组件从开始到消亡所经历的各种状态，就是一个组件的生命周期。共有八大生命周期，创建前/后,加载前/后，更新前/后，销毁前/后。
实例化一个vue对象

let app = new Vue({})

把新建的vue实例挂载到一个dom上

let app=new Vue({
    el:'#app'
})

这个#app元素是html中的DOM元素

<div id="app"></div>

vue实例中的所有东西都将在这个#app中显示
3，给vue实例中设置数据data

let app=new Vue({
    el:'#app',
    Data(){
        return{
            name:'xiaoxiao'
        }
    }
})

4,到这一步vue实例已经挂载到#app的dom元素上，数据data也有了，最后需要将vue实例中的数据绑定到试图上，也就是数据渲染。 

<div id="app">
<h1>hello {{name}}</h1>
</div>

ES6的学习总结

ECMAScript6简介

ES6是JavaScript语言的下一代标准，他的目标是使JavaScript能够编写复杂的大型应用程序成为企业级开发语言。ES6是JavaScript的一种规格，JavaScript是ES6的一种实现

let和const命令

ES6新增了let命令，用来声明变量，用法类似于var，但是所声明的变量只能在let命令所在的代码块内有效 

{let a=5;  
b=1;  
}  
a //error a is not defined  
b //1

for循环计数器就很适合使用let命令
let命令不存在变量提升，即变量不能在声明之前使用，不允许在相同作用域内重复声明同一变量
const命令声明一个只读的常量，一旦声明值就不能改变，所以const一旦声明就必须立即初始化，不能留到以后赋值const实际上保证，并不是变量的值不得改动而是变量指向的那个内存地址不能改动

ES6变量声明的六种方法

var function let const import class

Symbol

ES6引进了一种新的数据类型Symbol,表示独一无二的值，symbal函数不能使用new命令，否则会报错，这是因为生成的symbol是一个原始类型的值不是对象

生命不息学习不止

每天都是一个学习的过程，学习不能停止，今天复习了JavaScript的知识点，接下来要对今天的学习做个总结

JavaScript基础应用

match() 方法可以在字符串中检索指定的值，或找到一个或多个正则表达式的匹配

isNaN()判断是不是非数

变量是用于存储信息的容器

变量声明了但是没有赋值其值是undefined

可以通过将变量设置为空来清空变量

JS基本数据类型

Undefined,Null,Boolean,Number,String ECMAScript 2015新增：Symbol(创建后独一无二且不可变的数据类型)
object是JavaScript中所有对象的父对

JavaScript原型和原型链

每个对象都会在内部初始化一个属性就是prototype（原型），当我们访问一个对象的属性时如果这个对象内部不存在这个属性他就会去prototype里去寻找这个属性，这个prototype又会有自己的prototype这样一直找下去就是原型链的概念

关系：instance.constructor.prototype=instance.proto

JavaScript对象是通过引用来传递的，我们创建的每个新对象实体中并没有一份属于自己的原型副本。当我们修改原型时，与之相关的对象也会继承这一改变。当我们需要一个属性的时，Javascript引擎会先看当前对象中是否有这个属性， 如果没有的话，就会查找他的Prototype对象是否有这个属性，如此递推下去，一直检索到 Object 内建对象

JavaScript如何实现继承

1，构造函数继承 2，原型继承，3，实例继承，4，拷贝继承

this的理解

this总是指向函数的直接调用者（而非间接调用者）；
如果有new关键字，this指向new出来的那个对象；在事件中，this指向触发这个事件的对象，特殊的是，IE中的attachEvent中的this总是指向全局对象Window；

null和undefined的区别

null 表示一个对象是没有值的值，也就是值为空

undefined 表示变量声明了没有初始化未赋值的变量默认undefined

undefined不是一个有效的JSON，而null是

undefined的类型（typeof）是undefined null的类型是object；

一个空格引发的错误

今天成功搭建了自己的博客，真的是很开心，自己通过一步一步的步骤去实现，中间踩了很多坑，感谢小伙伴的悉心指导，经过几个小时的努力终于可以发布我的博客了，我想要把我踩过的坑跟大家分享一下避免大家以后遇到同样的问题去浪费时间。

我是用github+hexo搭建的博客

config.yml是一个很严格的文件，里面多一个空格或是少一个空格都会导致失败，我就是在文件中添加repository的时候前面没有加两个空格，在执行hexo d命令的时候一直出错,然后在GitHub page中一直没有master branch分支。提醒大家一下再打开config.yml文件的时候一定不要使用记事本这样会少踩很多坑，建议使用visual studio code
