一个空格引发的错误

今天成功搭建了自己的博客，真的是很开心，自己通过一步一步的步骤去实现，中间踩了很多坑，感谢小伙伴的悉心指导，经过几个小时的努力终于可以发布我的博客了，我想要把我踩过的坑跟大家分享一下避免大家以后遇到同样的问题去浪费时间。

我是用github+hexo搭建的博客

config.yml是一个很严格的文件，里面多一个空格或是少一个空格都会导致失败，我就是在文件中添加repository的时候前面没有加两个空格，在执行hexo d命令的时候一直出错,然后在GitHub page中一直没有master branch分支。提醒大家一下再打开config.yml文件的时候一定不要使用记事本这样会少踩很多坑，建议使用visual studio code
