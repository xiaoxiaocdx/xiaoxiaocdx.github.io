Posted on 2018-06-11   |  

Vue.js

搭建环境与运行项目

1, 想要运行一个vue项目首先需要安装git和node.js，node和git都是下载安装包直接下一步下一步安装就可以，记得如果这两个软件安装好后如果给他们改变了位置记得要从新配置环境变量啊。
2,然后安装 vue-cli 

npm install -g vue-cli

3,在你系统的某个盘里创建一个文件夹，为了存放以后你的vue项目代码我创建的是vue_gh，创建一个test项目 运行命令 

vue init webpack test

运行时一路回车等他结束会发现vue-gh文件夹中多了一个test文件夹
4，在dos命令行中进入项目文件夹 

cd vue-gh/test

5,安装依赖包 

npm install

6,启动项目 

npm run dev

Vue生命周期

一个组件从开始到消亡所经历的各种状态，就是一个组件的生命周期。共有八大生命周期，创建前/后,加载前/后，更新前/后，销毁前/后。
实例化一个vue对象

let app = new Vue({})

把新建的vue实例挂载到一个dom上

let app=new Vue({
    el:'#app'
})

这个#app元素是html中的DOM元素

<div id="app"></div>

vue实例中的所有东西都将在这个#app中显示
3，给vue实例中设置数据data

let app=new Vue({
    el:'#app',
    Data(){
        return{
            name:'xiaoxiao'
        }
    }
})

4,到这一步vue实例已经挂载到#app的dom元素上，数据data也有了，最后需要将vue实例中的数据绑定到试图上，也就是数据渲染。 

<div id="app">
<h1>hello {{name}}</h1>
</div>

MVVM模式

Vue实例是进入vue的第一步框架中的VM包含页面中的业务处理逻辑，数据模型等，他的生命周期中有多个事件的钩子，让我们在控制整个vue实例的过程中形成好的逻辑。Vue实例虽然没有完全遵循mvvm模型但受到了他的启发

Vue实际上对应的就是MVVM中的VM

View:是看得到的用到vue项目中来应该是模板，也就是用来挂载vue实例的一个DOM元素，通常在根目录的index.html中出现

Model：即模型或数据，就是想要显示到模型上的数据也是我么需要在程序生命周期中可能需要更新数据
MVVM模式种view和model是分开的但分开之后还要把他们联系在一起

vue指令

指令是带有v-前缀的特殊特性。
作用是当表达式的值改变时，将其产生的连带影响，响应式地作用于DOM

v-text:更新元素的textContext
v-html:更新元素的innerHTML
v-bind:当表达式的值改变时，将其产生的连带影响，响应式的作用与DOM,语法：v-bind:title="msg",简写： :title="msg"
v-on：绑定事件，绑定的事件从method中获取，语法 v-on:click="say"or v-on:click="say('参数'，$event)",简写：@click="say"
v-model 在表单元素上创建双向数据绑定，监听数据的输入事件以及更新数据
v-for: 基于源数据多次渲染元素或模板库
v-if: 条件渲染。根据表达式的值的真假条件，销毁或重建元素
v-show: 条件渲染 根据条件表达式的真假值，切换元素的display CSS属性
v-pre: 跳过这个元素和它的子元素的编译过程。可以用来显示原始 Mustache 标签。跳过大量没有指令的节点会加快编译
v-once:只渲染元素和组件一次。随后的重新渲染，元素/组件及其所有的子节点将被视为静态内容并跳过。这可以用于优化更新性能
