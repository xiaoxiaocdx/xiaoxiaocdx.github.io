gank项目总结

项目名称：gank手机端项目

项目搭建与运行

1，运用vue-cli脚手架搭建项目 

npm install -g vue-cli
vue init webpack vue-sample

2，安装依赖包 

npm install stylus --save
npm install stylus-loader --save
npm install vue-resource --save
npm install vuex --save

3,运行项目 

npm run dev

项目文件夹结构及用途

config—-配置信息
dist—-编译后的项目代码
src—-项目源码
asseets—存放项目图片
common—存放公共组件
components—存放组件
route—路由
App.vue—-跟组件

vue页面结构

项目思路以及框架搭建

要想写一个完整的项目首先需要把项目的整体框架搭建出来，然后再一点一点的往页面里面填充东西，这样一步一步的进行，当哪一点遇到问题时可以很容易的找到。
我写的这个gank项目只有三个页面，整体思路是一个标题栏（header），标题栏有一个图标点击一下会出现一个弹窗，标题栏下面时菜单栏（menu）,菜单栏的内容有 福利，Android，ios，下面就是这三个页面
因为不管点击那个页面标题栏和菜单栏都会出现，所以把他们两个设置成公共组件

export default { } 与 new Vue ({ })
new vue({}) 是为了实例化，创建一个Vue的实例 就是相当于创建一个根组件；
而export default {}是导出模块,供其他模块进行调用。一般都是导出一个组件，然后去父组件中定义引入就能使用

menu部分

menu部分要实现点击菜单栏自动切换的功能，
我自己定义了一个handelMenu方法，给菜单添加点击事件绑定该方法

type.HANDEL_CHANGE在common.js的mutations里面

页面部分

页面中数据的获取都是来自于 https://gank.io/api/data/福利/10/${this.page}
页面中主要是先把页面的布局整好然后写一个方法去获取数据
this.$http.get(https://gank.io/api/data/福利/10/${this.page}).then((response) => {}获取数据
获取到的数据用v-for指令渲染到页面上，v-for渲染的话必须在其父元素中进行渲染

v-for

v-for指令根据一组数组的列表进行渲染v-for item in items,item是自己起的名，items是源数组

弹窗

弹窗使用vuex 数据流切换 用v-on给弹窗添加点击事件
当点击弹窗按钮时执行showDialog()方法，show方法的值先设置为false意为隐藏，当点击时值编程true显示出来

##项目感想
这是我写的第一个简单的vue项目，通过对Es6和vue官方文档的了解，写了这个简单的项目，通过写这个项目还是有很多感触的，让我对vue框架又有了深层次的了解，如果只看文档和书不去练习想学好编程是不然容易的但是如果不看书和文档写项目又没有头绪，所以一定不要急于求成，只有一步一步的走下去才能走得更远。
.vue文件中文档规范要求的特别严格，一个空格或一个空行都可能引发错误，所以写的时候一定要细心,写项目中遇到了很多问题通过Google搜索和询问同学来解决问题。
